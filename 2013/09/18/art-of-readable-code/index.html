笔记

  • naming

    1. specific, not generic (ex. Fetch/Download instead of Get)

    2. More Important Details (ex. raw_ or _second)

    3. use capitalization, underscores, etc. in meaningful way

  • aesthetic

    1. “columns”

    2. “paragraphs”

    3. consistency

  • comment

    1. NOT comment for bad names - fix the name

    2. NOT comment for comment

    3. use input/output examples to clarify

    4. use inline-comments to clarify argument names (ex. calcDistance(/*time=*/10, /*speed*/=20);

    5. record flaws (ex. TODO, FIXME)

    6. compact

  • variables

    1. break down giant expressions

    2. avoid temporary variables, eliminate intermediate results

  • organization

    1. one task at a time

    2. logic - easier case first / more interesting case first

    3. extract unrelated subproblems; one function for one general task
