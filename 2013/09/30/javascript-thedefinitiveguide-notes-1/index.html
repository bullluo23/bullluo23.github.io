JavaScript, the Definitive Guide: Notes-1

  • lexical structure - the set of elementary rules that specifies how you write programs in that language
  • case sensitive -> HTML tag/attributes must in lower case
  • if a line break appeaer after return, break, continue, JavaScript always interpret that line break as semicolon
  • not-a-number value does not compare equal to any value, including itself
  • 0 and -0 are euqal, but 1/0 and 1/(-0) are not
  • Infinity and NaN are read/write-able in ECMAScript3 only
  • Number.POSITIVE_INFINITY, Number.NEGATIVE_INFINITY and Number.NaN are read-only
  • however, when actually run console.log(Infinity == Number.MAX_VALUE+1);, it will give False. See StackOverflow
  • global object - a regular Javascript object that the properties of this object are blobally defined symbol availiable to JavaScript program
  • evaluate to False: undefined, null, 0, -0, NaN, “”
  • difference between undefined and null:
    • undefined - predefined global variable, error
    • null - language keyword, expected, absence of value
  • objects are compared by reference instead of value
  • block scope - each block of code whithin {} has its own scope; variables are not visible outside of the block which they are declaired
  • function scope - var are visible within the function in which they are defined and whithinand function that are nested whithin that funciton
    • note the variables are availiable throughout the function
    • hoisting - JavaScript code behave as if all variables declared in a function are hoisted to top of function
  • delete variables as properties
    • true if the variable is deleted
    • cannot delete - built in core and client-side properties; user defined variable with var statement
    • note: dontDelete will actually successful deleted if the code is run in Firebug console, because in Firebug, code are fun as Eval code, instead of Global Code / Function Code. See this article
  • scope chain - a list or chain of objects that defines the variables that are in scope for that code
    • define function - stores the scope chain then in effect
    • invoke function - create new object to store its local variable and adds new object to the stored scope chain to create new, longer chain then represent the scope for that function invocation
  • order of evalute invocation expression:
    1. function expression; if not callable —> TypeError
    2. argument expression; produce a list of argument values
    3. argument values are assigned in order
    4. body of function is executed
  • instance of
    • LHS: object; RHS: identifies a class of object, function
  • void - evluate operand then discard value, return undefiend
    • to eliminate side effect
  • comma: statement1, statement2, ... , last_statement;
    • first statements will be execute, their value will be discard;
    • Racket: (begin statement1 statement2 ... last_statement)
  • eval()
    • pass a string - attemps to parse the string as JavaScript, throwing a SyntaxError if it fails; or evaluates the code and returns the value of the last expression or statement in the string or undefined if the last expression or statement had no value; if the string throws an exception, the eval() propagates that expression
    • eval() operate on the level of the scope it is currently in
  • global eval()
    • if define an alias of eval, i.e. var geval =eval;, most implementors wouldevlauate the string as top=level global code, and it could not use/modify variables local to the calling function (thus would not interfere with local optimizations)
    • this is called indirect eval, as the unqualified name “eval” is called direct eval
    • note: in ECMAScript 3, this would throw EvalError
